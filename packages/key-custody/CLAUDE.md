# @agenticprimitives/key-custody — Claude guide

## What this package owns
- `A2AKeyProvider` interface (envelope encryption + asymmetric signing + HMAC, all optional methods).
- Built-in providers: `LocalAesProvider` (dev), `AwsKmsProvider`, `GcpKmsProvider`.
- Built-in signers: `LocalSecp256k1Signer` (dev), `AwsKmsSigner`, `GcpKmsSigner`.
- Per-tool executor signers (`buildToolExecutorBackend(toolId, ...)`).
- viem adapter (`createKmsAccount(backend) → KMSSigner`).
- Relay-only signer (`getRelayOnlySigner` for Phase-B master-key safety).
- HMAC providers under `/mac` subpath (same backends, different threat model).
- AAD canonicalization (`canonicalContextBytes`).

## What this package does NOT own
- **Session lifecycle** — moved to [`delegation`](../delegation) per [ADR-0002](../../docs/architecture/decisions/0002-session-lifecycle-in-delegation.md). We expose envelope primitives only; `SessionManager` wires them.
- AAD **shape** decisions (caller provides; this package binds them in AES-GCM AAD + KMS EncryptionContext identically).
- Authority / policy / delegation mechanics → [`delegation`](../delegation), [`tool-policy`](../tool-policy).
- Consumer-app env parsing (we read documented env vars; consumers wire them).

## Vocabulary
**Owns:** `A2AKeyProvider`, `KmsAccountBackend`, `envelope encryption`, `AES-GCM`, `session-data-key`, `keyVersion`, `EncryptionContext`, `MAC provider`, `tool-executor signer`.
**Disambiguation (critical):**
- **`session-data-key`** here = the per-row AES-GCM key generated by `generateSessionDataKey`. This is **NOT** the `delegation.SessionRow` or the `identity-auth` JWT session — it's a crypto primitive named for its caller's use case.
- **`signer`** here = a concrete `KmsAccountBackend` implementation that becomes a `KMSSigner` (interface defined in [`identity-auth`](../identity-auth)) via `createKmsAccount`.
- **`AAD`** = additional authenticated data. The shape is the caller's choice (passed as `aadContext`); we canonicalize and bind.
See [`docs/architecture/vocabulary-map.md`](../../docs/architecture/vocabulary-map.md).
**Does not use:** `Delegation`, `Caveat`, `Enforcer`, `SessionManager`, `SessionRow`, `SessionPackage` (we encrypt opaque bytes), `JTI`, `RiskTier`, `evaluatePolicy`, `withDelegation`, MCP, OAuth, SIWE, passkey internals. See `capability.manifest.json:forbiddenTerms`.

## Drift triggers — STOP and route
- "Add a session lifecycle state machine, `SessionManager`, or `SessionStore`" — **STOP.** Belongs in [`delegation`](../delegation). [ADR-0002](../../docs/architecture/decisions/0002-session-lifecycle-in-delegation.md). We encrypt opaque bytes; we don't track lifecycle.
- "Define the `SessionPackage` shape" — **STOP.** `delegation` defines it. We accept the bytes via `generateSessionDataKey` / `decryptSessionDataKey`.
- "Verify a delegation, evaluate caveats, or mint a token" — **STOP.** Belongs in [`delegation`](../delegation).
- "Decide if a tool call is allowed" — **STOP.** Belongs in [`tool-policy`](../tool-policy).
- "Add an MCP route or transport adapter" — **STOP.** Belongs in [`mcp-runtime`](../mcp-runtime).

## Before you write code
- [ ] Is the change about envelope encryption, asymmetric signing, HMAC, or KMS backend selection?
- [ ] If I'm touching "session" in a name, am I sure I mean `session-data-key` (right) and not the `delegation.SessionRow` (wrong place)?
- [ ] Am I treating session payloads as **opaque bytes** with caller-supplied AAD, not parsing them?
- [ ] If I'm adding a backend, does it implement `A2AKeyProvider` and respect the production guard (`local-aes` refuses in production)?
- [ ] Did I update `specs/203-key-custody.md` if the public API changed?

## Security invariants (DO NOT BREAK)
- **No production fallback.** AWS/GCP backends MUST fail-closed on outage. No silent local fallback.
- **Production guard.** `local-aes` and any `*_PRIVATE_KEY` env MUST throw at boot when `NODE_ENV=production`.
- **Master-key separation.** Master signer, per-tool executor, and session-data-key wrapping MUST be distinct IAM-scoped KMS keys (operational; we validate at boot).
- **Relay-only master.** `getRelayOnlySigner` MUST throw on any `signMessage` / `signUserOp` call.
- **AAD trip-wire.** AES-GCM AAD and KMS EncryptionContext MUST be bound identically; tampering trips both.
- **Audit trail.** Every `Decrypt` and signing op emits an audit row with `keyVersion`, hashed sessionId, optional toolId/actionId. **Raw sessionId MUST NEVER be logged.**

## Common task routing
- Adding a new KMS backend → implement `A2AKeyProvider` in `src/providers/<backend>.ts`, register in `buildKeyProvider`, add subpath export.
- Adding a new audit field → `src/audit.ts`; coordinate with `delegation` (it consumes the audit context shape).
- Crypto changes → security review required; AAD invariants in particular.

## Capabilities this package participates in
