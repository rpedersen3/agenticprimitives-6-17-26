# Solhint config rationale (R9.1)

`.solhint.json` is intentionally a strict-schema config (no sibling `//`
comment keys allowed — Solhint validates the top-level shape). The rationale
that belongs in the file lives here instead.

## Why not `extends: "solhint:recommended"`

The `solhint:recommended` bundle includes style rules
(`max-line-length`, `func-name-mixedcase`, `var-name-mixedcase`,
`ordering`, `quotes`, `func-visibility` with `ignoreConstructors: false`,
etc.) that would force a large mechanical churn of the existing
contracts codebase without proportional security gain.

R9.1 opts INTO the security-focused subset only. Style alignment is a
follow-up wave (R9.6+) after the high-ROI gates land.

## Why some "security-looking" rules are OFF

| Rule | Setting | Reason |
|---|---|---|
| `not-rely-on-time` | `off` | Timelocks (`CustodyPolicy.eta`) and `TimestampEnforcer` caveats intentionally consume `block.timestamp`. Those surfaces are locked by invariant tests, not a blanket lint ban. |
| `reason-string` | `off` | Custom errors replaced revert strings repo-wide (Solidity 0.8.4+); the rule fires false positives on `require(..., "...")` patterns that don't exist here. |
| `avoid-low-level-calls` | `off` | `AgentAccount.execute` + the enforcers intentionally use `address.call` / `delegatecall` for ERC-4337 / ERC-7579 dispatch. The replay/authority surface is covered by the auth-closure tests + Slither's `controlled-delegatecall` detector. |
| `no-inline-assembly` | `off` | `WebAuthnLib` / P-256 verification uses targeted assembly for performance; `AgentAccount.executeFromValidator` uses assembly for ERC-1271 magic-value return. Reviewed paths only. |
| `no-console` | `warn` | Foundry scripts use `console.log` for deploy/migration output; that's intentional. The warn level surfaces accidental leaks into `src/`. |

## Why these rules ARE on

| Rule | Setting | Reason |
|---|---|---|
| `compiler-version` `^0.8.28` | `error` | Repo-wide pragma; drift breaks via-IR + storage-layout determinism. |
| `avoid-tx-origin` | `error` | `tx.origin` has no legitimate use in this codebase (custody-policy + delegation are all `msg.sender`-bound). |
| `avoid-suicide` | `error` | `selfdestruct` is post-Cancun a no-op for stored balances but the *opcode* is still a footgun in upgradeable contexts. Blanket ban. |
| `avoid-sha3` | `error` | Use `keccak256`, not the legacy `sha3` alias. |
| `avoid-throw` | `error` | `throw` was deprecated in Solidity 0.4.x; legacy hits should never re-enter the codebase. |
| `check-send-result` | `error` | `address.send` ignores reverts; we want explicit `call` + result check. |
| `state-visibility` | `error` | Implicit-default visibility on state vars is a known footgun; require explicit `private`/`internal`/`public`. |
| `no-complex-fallback` | `error` | `fallback()` should never do work other than module-dispatch (ERC-7579); a complex fallback is a routing bug. |

## How to run locally

```bash
cd packages/contracts
npx solhint 'src/**/*.sol'
```

CI invokes the same command via `pnpm check:contracts-lint` in the
`security.yml` workflow (PR-blocking).
