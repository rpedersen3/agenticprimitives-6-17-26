# Vocabulary Map

**Purpose:** single source of truth for cross-package terminology. When the same word means different things in different packages, this file is the disambiguator. Every package's `CLAUDE.md` references this file when its vocabulary touches a neighbor's.

The drift mode this prevents: an agent working in package X confuses X's concept of "session" with Y's concept of "session" and writes code that violates Y's invariants.

---

## 1. Ambiguous terms across packages

Each row: a term, the package that owns it in that sense, and what it means there. If you see the term in code or docs, **read the row matching your current package**, not a neighbor's.

| Term | Owning package | Meaning |
| --- | --- | --- |
| **session** | `delegation` | A `SessionRow` binding a `Delegation` to a session-signing keypair. Lifecycle: `pending` → `active` → `revoked`/`expired`. Managed by `SessionManager`. |
| **session** | `identity-auth` | A JWT-cookie session bound to a user identity (claims `sub`, `walletAddress`, `smartAccountAddress`, `via`). Lives in `agentic-session` cookie. |
| **session-data-key** | `key-custody` | A per-row data key generated by `generateSessionDataKey`. AES-GCM key material for envelope encryption. **Not** the session above — this is the key used to encrypt the payload. |
| **signer** | `identity-auth` | The abstract `Signer` interface (`PasskeySigner` / `EOASigner` / `KMSSigner`). What a delegation gets signed with. |
| **signer** | `key-custody` | A concrete `KMSSigner` implementation produced by `createKmsAccount(backend)`. The viem-shaped object that does the signing. |
| **account** | `agent-account` | A deployed ERC-4337 smart contract account. Has a deterministic CREATE2 address. Implements ERC-1271. |
| **account** | `identity-auth` | The subject of an `AuthenticatedUser` claim. May or may not be a smart account; could be just a `walletAddress`. |
| **tool** | `tool-policy` | A classified tool definition produced by `declareTool(def, classification)`. Transport-agnostic; could be MCP, A2A, or LangChain. |
| **tool** | `mcp-runtime` | A specific MCP tool registered with `@modelcontextprotocol/sdk` and wrapped with `withDelegation`. |
| **envelope** | `key-custody` | AES-GCM envelope encryption of a payload (the `encryptedPackage` + `iv` + `encryptedDataKey` triple). |
| **envelope** | `mcp-runtime` | The inter-service HTTP request envelope with an HMAC over the body. |
| **policy** | `tool-policy` | A decision produced by `evaluatePolicy(ctx)`. One of `allow` / `deny` / `requires-consent`. |
| **policy** | other packages | Avoid using "policy" outside `tool-policy`. If you need to express constraints in another package, call them **caveats** (delegation), **classification** (tool-policy via metadata), or **invariants** (security boundary). |
| **principal** | `delegation` | The verified `delegator` address from delegation verification. The "who" downstream code can trust. |
| **principal** | `mcp-runtime` | Same value, passed into the wrapped tool handler as `principal: Address`. Originates in `delegation`. |
| **caveat** | `delegation` | A `Caveat` in an EIP-712 `Delegation` struct. Restricts authority. Eight on-chain enforcers + three off-chain sentinels. |
| **caveat** | other packages | **Don't use.** If you need a restriction, it either becomes a caveat in `delegation` (and you build it via `buildCaveat`) or it becomes classification metadata in `tool-policy`. |
| **enforcer** | `delegation` | A `Caveat.enforcer` field — either an on-chain contract address or an off-chain sentinel address (`keccak256('urn:...')[0:20]`). |
| **enforcer** | other packages | Don't use. |
| **risk tier** | `tool-policy` | One of `low` / `medium` / `high` / `critical` applied to a classified tool. Used to clamp delegation TTLs and to gate consent prompts. |
| **risk tier** | `delegation` | Consumed via `tool-policy.requiredCaveatsForRiskTier` when an issuer builds caveats. Don't redefine the levels. |
| **audit context** | `key-custody` | `{ toolId?, sessionId?, actionId? }` passed to signing/decrypt for logging. Hashed sessionId only — raw never logged. |
| **audit context** | `mcp-runtime` | Broader concept: caller identity + classification + delegation hash at the request boundary. Different fields. |

## 2. Rule of thumb: which package owns a word

When you're tempted to introduce a term in package X, ask:

1. **Does another package already use this word for a different concept?** Check the table above. If yes, either pick a different word (preferred) or qualify it (`a2aSession` vs `userSession`).
2. **Is this concept actually owned by another package?** If yes, route there. Don't create a parallel concept here.
3. **Does this term have a stable, narrow definition in your package?** If it's loose, name it more specifically.

## 3. Package-specific vocabulary lists

What each package owns and what it deliberately does NOT use. (Reproduced from each package's `capability.manifest.json:forbiddenTerms` and `CLAUDE.md:Vocabulary` for cross-reference.)

### `@agenticprimitives/types`
- **Owns:** `Address`, `Hex`, `ChainId`, `BrandedId`.
- **Does not use:** anything domain-specific. No `User`, `Session`, `Tool`, `Delegation`. Types-only leaf.

### `@agenticprimitives/identity-auth`
- **Owns:** session (JWT, cookie-bound), claims, CSRF, auth methods (passkey / SIWE / Google), `Signer` interfaces.
- **Does not use:** `Delegation`, `Caveat`, `Enforcer`, KMS backends, MCP, JTI, envelope encryption, session-data-key.

### `@agenticprimitives/agent-account`
- **Owns:** smart account, factory, CREATE2 salt, ERC-1271, UserOp building, EntryPoint, bootstrap signer.
- **Does not use:** `Delegation`, `Caveat`, KMS backends, `SessionManager`, MCP, policy, tool classification.

### `@agenticprimitives/delegation`
- **Owns:** `Delegation`, `Caveat`, `Enforcer`, EIP-712 hashing, `DelegationToken`, `SessionManager` (session = SessionRow), `SessionStore`, JTI semantics.
- **Does not use:** KMS provider implementations, envelope encryption algorithms, AAD shape decisions (we pass; key-custody binds), MCP transport, OAuth, passkey internals, policy taxonomy.

### `@agenticprimitives/key-custody`
- **Owns:** `A2AKeyProvider`, envelope encryption, AES-GCM, KMS backends (local/AWS/GCP), HMAC providers, signers (master + per-tool), session-data-key, AAD canonicalization.
- **Does not use:** `Delegation`, `Caveat`, `Enforcer`, `SessionManager` / session lifecycle, JTI, policy, MCP, OAuth.

### `@agenticprimitives/tool-policy`
- **Owns:** classification taxonomy (`@sa-*` tags), `RiskTier`, `ToolClassification`, `declareTool`, exact-call DSL, `evaluatePolicy`, `lintClassification`.
- **Does not use:** any protocol transport (MCP / A2A / LangChain / Vercel), `Delegation` mechanics (only consumed as type), KMS, envelope encryption, JWT, sessions.

### `@agenticprimitives/mcp-runtime`
- **Owns:** `withDelegation`, `withCrossDelegation`, JTI stores, `declareResource`, MCP request envelope (HMAC), bridge from `delegation` + `tool-policy` to MCP tool errors.
- **Does not use:** passkey, SIWE, OAuth, smart account factory, ERC-1271 internals, KMS backend implementations, classification taxonomy (imported from tool-policy), caveat evaluator (imported from delegation).

---

## 4. When in doubt

If a term doesn't appear in this map and feels ambiguous: that's a signal to either rename it more specifically in your package, or add a row here documenting the choice. Don't quietly introduce a meaning that conflicts with a neighbor.
