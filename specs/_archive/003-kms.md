# Spec 003 — `@agenticprimitives/kms`

**Capability:** Pluggable KMS abstraction for agent session keys, layered on delegation. Provides envelope encryption for session state, key-bound signers for tokens/UserOps, and HMAC providers for inter-service authentication.
**Status:** v0 draft · 2026-05-19
**Reference implementation:** `smart-agent/packages/sdk/src/key-custody/*`; `smart-agent/apps/a2a-agent/src/auth/{key-provider,a2a-signer,encryption,mac-provider,sign-outbound}.ts`.

---

## 1. Goal

Provide a uniform abstraction that lets an A2A agent (or any service) do four things without holding plaintext key material:

1. **Envelope-encrypt** a session package (which contains a session private key + a delegation) at rest, bound to AAD that includes session-id, account, chain, expiry.
2. **Sign** tokens and on-chain transactions using a key the KMS holds (a "session signer" or a per-tool executor signer), without ever exfiltrating it.
3. **Generate HMACs** for inter-service envelopes (web↔a2a, a2a↔mcp), again with the secret never leaving the KMS boundary.
4. **Swap backends** — local-AES (dev), AWS KMS, GCP KMS — by changing env, with a hard guard against using dev backends in production.

This package layers on `@agenticprimitives/delegation`: the session keys it manages are precisely the `delegate` keys in delegations issued by user smart accounts.

## 3. Core interfaces

```ts
export interface A2AKeyProvider {
  readonly keyVersion: string;          // e.g. 'local-v1', 'aws-kms:abc-uuid'

  /** Generate a fresh data key; return plaintext + KMS-wrapped blob. */
  generateSessionDataKey(input: {
    aadContext: Record<string, string>;
  }): Promise<{
    plaintextDataKey: Uint8Array;
    encryptedDataKey: Uint8Array;
    keyId: string;
    keyVersion: string;
  }>;

  /** Decrypt a wrapped data key. AAD must match exactly what was used at wrap time. */
  decryptSessionDataKey(input: {
    encryptedDataKey: Uint8Array;
    aadContext: Record<string, string>;
    keyId: string;
    keyVersion: string;
  }): Promise<Uint8Array>;

  /** Asymmetric secp256k1 signing — for master / per-tool signers. Optional. */
  signA2AAction?(input: {
    digest: Uint8Array;
    auditContext?: { toolId?: string; sessionId?: string; actionId?: string };
  }): Promise<{ signature: Uint8Array; keyId: string; signerAddress: Address }>;

  /** HMAC-SHA-256 for inter-service envelopes. Optional. */
  generateMac?(input: {
    canonicalMessage: Uint8Array;
    service: string;
    audience: string;
  }): Promise<{ mac: Uint8Array; keyId: string }>;
}
```

The interface uses **optional methods** because not every backend implements every capability (e.g., `local-aes` may not implement `signA2AAction`; a dedicated signer backend is paired with it).

## 4. Session package model

A "session" in this package = the row that binds an issued delegation to its session signing key.

```ts
export interface SessionRow {
  id: string;                           // 'sa_<uuid>'
  accountAddress: Address;              // smart account (delegator)
  sessionKeyAddress: Address;           // pubkey of the ephemeral signer
  status: 'pending' | 'active' | 'revoked' | 'expired';
  encryptedPackage: Uint8Array;         // AES-GCM ciphertext of SessionPackage
  iv: Uint8Array;
  encryptedDataKey: Uint8Array;         // KMS-wrapped data key
  keyVersion: string;
  expiresAt: string;                    // ISO; clamped per risk-tier
  variant?: 'A' | 'B';                  // see spec 002 §5
  createdAt: string;
  revokedAt?: string;
}

export interface SessionPackage {
  sessionPrivateKey: Hex;
  delegation: Delegation;                // from @agenticprimitives/delegation
}
```

AAD bound at encryption time:
```
{
  session_id_h: sha256(sessionId)[0..16],  // never raw sessionId
  account_address: <lowercase>,
  chain_id: '<number>',
  expires_at: <ISO>,
  key_version: <provider.keyVersion>,
}
```

Tampering with any of these → KMS context mismatch (AWS/GCP) AND AES-GCM tag failure (defense in depth).

Smart-agent ref: `apps/a2a-agent/src/auth/encryption.ts:195-383`, `apps/a2a-agent/src/db/schema.ts:33-86`.

## 6. Security boundaries (preserved from smart-agent)

1. **No production fallback.** AWS/GCP backends fail-closed if KMS is unreachable — there is no local fallback path.
2. **Production guard.** `local-aes` and any `*_PRIVATE_KEY` env throw at boot when `NODE_ENV=production` (or `A2A_KMS_BACKEND` is a managed backend).
3. **Master-key separation.** The master signer key, per-tool executor keys, and session-data-key wrapping keys are distinct IAM-scoped KMS keys.
4. **Relay-only master.** In Phase B, the master signer's role is reduced to "broadcaster only" — it cannot sign on the user's behalf. `getRelayOnlySigner()` returns a signer that throws on any `signMessage`/`signUserOp` call (`apps/a2a-agent/src/auth/a2a-signer.ts:215-254`).
5. **AAD trip-wire.** AES-GCM AAD and KMS EncryptionContext are bound identically; any tampering trips both checks.
6. **Audit trails.** Every `Decrypt` and signing operation emits an audit row including `keyVersion`, hashed sessionId, optional toolId/actionId. Raw sessionId never logged.
7. **CloudTrail / Cloud Audit Logs** carry KMS `GenerateDataKey`/`Decrypt` calls with `EncryptionContext` matching the AAD — gives a forensic trail without exposing secrets.

## 7. Layering on delegation

This package is delegation-aware but not delegation-internals-aware:

- It treats `SessionPackage.delegation` as an **opaque payload** at the encryption layer — just bytes to encrypt.
- It exposes the `sessionPrivateKey` as a `LocalAccount` via `createKmsAccount()` so the delegation package can call `mintDelegationToken(claims, account.signMessage)` without ever seeing the private key in JS memory longer than necessary.
- Session-key rotation = mint a new session package; old key is never re-used.

The integration boundary is one line in the a2a-agent route layer:
```ts
const pkg = await decryptSessionPackage(row, meta, provider);
const sessionAccount = privateKeyToAccount(pkg.sessionPrivateKey);
const { token } = await mintDelegationToken({...claims, delegation: pkg.delegation},
                                            sessionAccount.signMessage);
```

## 8. Test plan (v0)

- Unit: AAD bind/unbind, AES-GCM round trip, KMS provider mocks for AWS/GCP (sdk-mock or local stub), production guard rejection.
- Integration: a real `LocalAesProvider` + `LocalSecp256k1Signer` issuing → mint → verify round trip with the delegation package.
- Negative: tampered ciphertext (flip a byte → tag failure), tampered AAD (KMS context mismatch), wrong `keyVersion` (no decrypt).
- Conformance (against AWS/GCP): provider trip-wire test in CI behind explicit env opt-in — does NOT run by default to avoid charging real KMS calls.
